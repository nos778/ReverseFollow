# ReverseFollow 反向跟单策略

`ReverseFollow.py` 是一个无限易 PythonGo 反向跟单策略，用于在同一客户端内，让目标账号按照源账号的反方向持仓。

策略不是逐笔复制成交，而是根据源账号当前持仓计算目标账号应有的反向仓位，并在收到合约 tick 后自动调仓。

## 核心逻辑

```text
理论目标多头 = int(源账号空头 * 反向系数)
理论目标空头 = int(源账号多头 * 反向系数)
```

例如：

| 源账号持仓 | 反向系数 | 目标账号理论持仓 |
| --- | ---: | --- |
| 多头 10 手 | 1.0 | 空头 10 手 |
| 空头 8 手 | 1.0 | 多头 8 手 |
| 多头 6 手 | 0.5 | 空头 3 手 |
| 空头 3 手 | 2.0 | 多头 6 手 |

当目标账号实际持仓和理论持仓不一致时，策略会自动发单调整。

## 主要功能

- 支持指定源账号和目标账号。
- 支持指定交易所、合约、投机套保标志。
- 支持设置反向系数。
- 支持限制单次最大跟单手数。
- 支持对手价或最新价发单。
- 支持设置滑点。
- 支持开启或关闭自动交易。
- 存在未完成挂单时，不会重复发新单。

## 参数说明

| 参数 | 说明 |
| --- | --- |
| `exchange` | 交易所代码 |
| `instrument_id` | 合约代码 |
| `source_investor` | 源账号，读取该账号持仓 |
| `target_investor` | 目标账号，策略向该账号发单 |
| `hedgeflag` | 投机套保标志，默认 `1` |
| `reverse_ratio` | 反向系数，默认 `1.0` |
| `max_follow_volume` | 单次最大跟单手数 |
| `price_mode` | 发单价格模式，`opponent` 为对手价，`last` 为最新价 |
| `slippage_ticks` | 滑点跳数 |
| `market` | 是否市价单 |
| `enable_trading` | 是否允许自动交易 |

## 调仓顺序

每次收到指定合约 tick 后，策略按以下顺序判断：

1. 目标多头过多：卖出平多。
2. 目标空头过多：买入平空。
3. 目标多头不足：买入开多。
4. 目标空头不足：卖出开空。

每次发单手数为：

```text
min(仓位差额, max_follow_volume)
```

如果仓位差较大，会分多次逐步调整。

## 使用步骤

1. 将 `ReverseFollow.py` 放入无限易 PythonGo 策略目录。
2. 在 PythonGo 中加载 `ReverseFollow` 策略。
3. 填写交易所、合约、源账号、目标账号等参数。
4. 首次运行建议先设置 `enable_trading=False`，只观察持仓和理论仓位是否正确。
5. 确认无误后，再设置 `enable_trading=True` 开启自动交易。
6. 实盘前建议先用小手数或模拟盘测试。

## 注意事项

- 本策略是持仓对齐型反向跟单，不是成交逐笔复制。
- 策略依赖 tick 触发，没有行情 tick 时不会主动调仓。
- 如果存在未完成挂单，策略会等待挂单结束后再继续判断。
- 如果持仓读取失败，代码会按 0 手处理，实盘前必须确认持仓查询稳定。
- 源账号和目标账号不要填同一个账号。
- `reverse_ratio` 计算后会使用 `int()` 取整，可能产生小数舍弃。
- 策略没有内置止损、止盈、最大亏损、交易时段过滤等风控。

## 风险提示

自动交易存在风险。使用前请确认账号权限、资金、保证金、行情、合约参数和交易接口均正常。任何拒单、滑点、延迟、网络异常或持仓读取异常，都可能导致实际持仓与预期不一致。
